Every Layout - The Center, Vertical, By Flex, By Margin

This serie shows how to vertical center an element by applying a flex box on the parent element plus applying margins on the flex items.

The first example, centerReset shows an application of tw-center-v-byflex-bymar-4, hence the element shrinks to it's instrinsic content and leaving application for the margins, in effect centering it's content.

The second example, minimal-containerReset , additionally appliestw-stack-byflex-natural-meto the container of the element defining the tw-center-h-byflex-bymar-4. Hence the contained elements do not expand, leaving no application for the margins. Still, the paddings to the top and bottom remain.

The third example, sufficient-textonlyReset , just shows the minimal sufficent application of tw-center-h-byflex-bymar-4 for a text-only node. The Plugins so far normally provide the auto margins via prefabricated CSS selectors to the child nodes, but as this is a text-only node, we have to supply a hint with tw-center-v-byflex-bymar-me to the tw-center-h-byflex-bymar-4 itself.

The fourth example, sufficient-elmonlyReset , at last, just shows the minimal sufficent application of tw-center-h-byflex-bymar-4 for a element-only node.

This element will be replaced by any element iff a root element of id usage-example exists and having an attribute of for-code-example. Then any node having an attribute of for-code-example is selected, formatted and highlighted here.

In the first and second example, the element defining the tw-center-h-byflex-bymar-4 is colored with tw-bg-purple-400. It's container is colored with tw-bg-yellow-100 - but because how the margins work inside a flex box, this container is not visible. The content element is colored with tw-bg-blue-500. To show these colored effects, the DOM of these examples is contrived into multiple HTML elements. In practice, just a single element with tw-center-h-byflex-bymar-4 would be sufficient - this is what the third and fourth example shows.
